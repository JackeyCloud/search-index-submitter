# 搜索引擎一键提交工具：用户配置与使用指南

> 适用版本：`Search Index Submitter 0.1.x`<br>
> 文档更新：2026 年 6 月 18 日<br>
> 适用对象：站长、独立开发者、SEO 运营人员、建站公司及批量维护多个网站的团队

---

## 1. 这款工具能做什么

本工具用于把新网站、新页面和 Sitemap 统一提交给多个搜索平台，减少在不同站长平台之间来回操作。

当前支持：

| 平台 | 工具执行的操作 | 是否需要用户配置凭据 |
|---|---|---|
| IndexNow | 批量通知 Bing、Yandex 等参与平台 URL 已新增或更新 | 需要 IndexNow Key |
| 百度搜索 | 调用百度“普通收录”API 推送 URL | 需要百度站点 token |
| Google | 查询 URL 收录状态，并通过 Search Console API 提交 Sitemap | 需要 Google OAuth 客户端 JSON |
| Bing | 查询 URL 状态，并调用 Bing Webmaster API 提交 URL | 需要 Bing Webmaster API Key |
| Yandex | 通过 Yandex Webmaster API 提交 Sitemap | 需要 Yandex OAuth Token |
| 360 搜索 | 显示人工提交入口和操作提示 | 暂无稳定公开 API |
| 神马搜索 | 显示人工提交入口和操作提示 | 暂无稳定公开 API |

### 重要说明

1. “提交成功”只表示平台已接收通知，不等于一定收录。
2. 是否抓取、收录和参与排名，最终由搜索引擎决定。
3. 网站必须能被公网访问，并且没有通过 `robots.txt`、登录验证或防火墙阻止搜索引擎。
4. Google 已停用旧的 Sitemap Ping 接口；普通网站不能把 Google Indexing API 当成通用收录接口。
5. 工具只有在平台明确确认 URL 已收录时才会跳过。查询失败或无法确认时仍会提交，避免漏提。

## 2. 开始配置前的准备工作

在申请任何平台凭据之前，先检查网站本身。

### 2.1 网站必须可以正常访问

用浏览器打开：

```text
https://你的域名/
```

确认：

- 页面返回正常内容，不是 404、502 或维护页面；
- HTTPS 证书有效；
- 手机和电脑网络都能访问；
- 不需要输入账号密码才能看到主要页面；
- 网站不会把搜索引擎爬虫全部拦截。

### 2.2 准备 robots.txt

推荐地址：

```text
https://你的域名/robots.txt
```

基础示例：

```text
User-agent: *
Allow: /

Sitemap: https://你的域名/sitemap.xml
```

不要出现以下配置，除非你确实要禁止整个网站被抓取：

```text
User-agent: *
Disallow: /
```

### 2.3 准备 Sitemap

推荐地址：

```text
https://你的域名/sitemap.xml
```

用浏览器打开该地址，确认返回 XML 内容，而不是登录页或 404 页面。

最小示例：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/</loc>
    <lastmod>2026-06-18</lastmod>
  </url>
</urlset>
```

### 2.4 统一网站版本

下面这些地址在站长平台中可能被视为不同站点：

```text
http://example.com/
https://example.com/
https://www.example.com/
https://example.com/path/
```

建议确定唯一正式版本，并做好 301 跳转与 canonical：

```html
<link rel="canonical" href="https://example.com/当前页面">
```

软件中输入的地址、Sitemap 地址和站长平台验证的站点应保持一致。

## 3. 安装和启动软件

### 3.1 macOS

第一次运行：

```bash
cd search-index-submitter
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip setuptools
.venv/bin/pip install -r requirements.txt
```

以后可以双击：

```text
启动工具.command
```

### 3.2 Windows

先安装 Python 3.9 或更高版本，并在安装界面勾选“Add Python to PATH”。

在项目目录打开 PowerShell：

```powershell
py -m venv .venv
.venv\Scripts\python.exe -m pip install --upgrade pip setuptools
.venv\Scripts\pip.exe install -r requirements.txt
.venv\Scripts\python.exe run.py
```

### 3.3 Linux

```bash
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip setuptools
.venv/bin/pip install -r requirements.txt
.venv/bin/python run.py
```

### 3.4 配置文件保存位置

平台凭据默认保存在：

```text
~/.search-index-submitter/config.json
```

macOS 和 Linux 会尝试把文件权限设置为仅当前用户可读写。请不要：

- 把该文件上传到 GitHub；
- 发给其他人；
- 在客服截图中展示完整 token 或 API Key；
- 放进网站公开目录。

## 4. 软件界面如何填写

### 4.1 批量网址输入框

每行输入一个域名或完整 URL：

```text
example.com
https://www.example.org/
https://shop.example.net/products/new
```

工具会自动：

- 给裸域名补充 `https://`；
- 按站点分组；
- 去除重复 URL；
- 尝试从 `robots.txt` 发现 Sitemap；
- 尝试常见的 `/sitemap.xml` 等路径。

### 4.2 Sitemap 输入框

一般留空即可自动发现。

单个站点可以填写完整地址：

```text
https://example.com/sitemap.xml
```

批量站点可以使用模板：

```text
https://{host}/sitemap.xml
```

### 4.3 “提交前查重”选项

建议保持勾选。

- Google：使用 URL Inspection API 查询；
- Bing：使用 Webmaster GetUrlInfo 查询；
- 确认已收录：从本次提交中移除；
- 明确未收录：正常提交；
- 无凭据、接口受限或状态不明确：显示“无法确认”，仍然提交。

### 4.4 预检与正式提交

第一次使用某个站点时，先点“预检”。

预检会检查：

- URL 是否有效；
- Sitemap 是否能发现；
- 平台凭据是否已经配置；
- 平台是否支持自动查询或提交；
- 本次准备处理多少 URL。

确认结果无误后，再点“开始一键提交”。

## 5. IndexNow Key 获取与配置

IndexNow 是最容易配置的平台方案之一。一份请求可以通知多个参与 IndexNow 的搜索引擎，但不包含 Google 和百度。

### 5.1 生成 Key

Key 建议使用 32 位十六进制字符串。

Windows PowerShell：

```powershell
[guid]::NewGuid().ToString("N")
```

macOS 或 Linux：

```bash
openssl rand -hex 16
```

示例：

```text
cd7c10e8b36050cfad3f045df2126986
```

### 5.2 在网站根目录创建验证文件

假设 Key 是：

```text
cd7c10e8b36050cfad3f045df2126986
```

创建文件：

```text
cd7c10e8b36050cfad3f045df2126986.txt
```

文件内容也必须是同一个 Key：

```text
cd7c10e8b36050cfad3f045df2126986
```

把文件上传到网站根目录，最终必须可以访问：

```text
https://example.com/cd7c10e8b36050cfad3f045df2126986.txt
```

浏览器打开后应只显示 Key，并返回 HTTP 200。

### 5.3 软件内配置

打开“平台凭据”，填写：

```text
IndexNow Key：cd7c10e8b36050cfad3f045df2126986
Key 文件 URL：https://example.com/cd7c10e8b36050cfad3f045df2126986.txt
```

如果 Key 文件就在当前站点根目录，“Key 文件 URL”通常可以留空。

### 5.4 批量站点注意事项

可以让多个站点使用同一个 Key，但每一个域名都必须能在自己的根目录访问对应 Key 文件。

例如：

```text
https://site-a.com/相同Key.txt
https://site-b.com/相同Key.txt
```

### 5.5 常见错误

| 状态 | 原因 | 处理方法 |
|---|---|---|
| HTTP 400 | 请求格式错误 | 检查 URL 和 Key 格式 |
| HTTP 403 | Key 文件无法验证 | 确认 Key 文件地址返回 200，文件名和内容一致 |
| HTTP 422 | URL 不属于提交的主机 | 不要把不同域名混进同一个站点请求 |
| HTTP 429 | 请求过于频繁 | 等待一段时间后再提交，不要反复点击 |

官方资料：[IndexNow API 文档](https://www.indexnow.org/documentation)

## 6. 百度普通收录 token 获取与配置

百度 token 通常与已验证站点相关。多个站点可能需要分别配置 token。

### 6.1 注册和登录

1. 打开[百度搜索资源平台](https://ziyuan.baidu.com/)。
2. 使用百度账号登录。
3. 完成账号要求的身份信息或安全验证。

### 6.2 添加并验证网站

1. 进入“用户中心”或“站点管理”。
2. 点击“添加网站”。
3. 填写网站协议和域名，例如：

```text
https://example.com
```

4. 选择平台提供的验证方式，常见方式包括：
   - DNS 验证；
   - HTML 文件验证；
   - HTML 标签验证。
5. 按页面提示完成验证。

推荐 DNS 验证。网站改版或更换服务器后通常不容易丢失。

### 6.3 获取普通收录 API token

1. 进入已经验证的网站。
2. 找到“资源提交”或“普通收录”。
3. 打开“API 提交”。
4. 页面会显示类似下面的接口地址：

```text
https://data.zz.baidu.com/urls?site=https://example.com&token=你的token
```

5. 只复制 `token=` 后面的内容，不要复制整个接口地址。

### 6.4 软件内配置

只有一个站点时，把 token 填入：

```text
百度 token
```

多个站点使用不同 token 时，在“百度多站 token”中填写：

```text
site-a.com=tokenA,site-b.com=tokenB,www.site-c.com=tokenC
```

注意：

- 只填写域名，不要填写 `https://`；
- `www.example.com` 和 `example.com` 应按实际站点分别填写；
- 默认 token 用于映射表中没有单独配置的站点；
- 百度每天的可提交数量以平台返回值为准。

### 6.5 常见错误

| 提示 | 原因 | 处理方法 |
|---|---|---|
| site error | 输入网站与百度验证站点不一致 | 检查协议、主域名和 `www` |
| token 无效 | token 复制错误或已经变化 | 回到 API 提交页重新复制 |
| not same site | URL 不属于该站点 | 按域名拆分批次 |
| 今日剩余为 0 | 当日额度用完 | 次日再提交，不要频繁重试 |

官方入口：[百度搜索资源平台普通收录](https://ziyuan.baidu.com/linksubmit/index)

## 7. Google Search Console OAuth 配置

Google 配置步骤最多，分为“验证网站”和“创建 OAuth 客户端”两部分。

### 7.1 在 Search Console 验证网站

1. 打开 [Google Search Console](https://search.google.com/search-console)。
2. 登录准备管理网站的 Google 账号。
3. 点击“添加资源”。
4. 推荐选择“网域”资源并填写：

```text
example.com
```

5. 按提示在域名 DNS 中添加 TXT 记录。
6. 等待 DNS 生效后点击验证。
7. 验证成功后，在 Search Console 中提交一次 Sitemap：

```text
https://example.com/sitemap.xml
```

如果无法修改 DNS，也可以选择“网址前缀”资源，但必须确保协议和域名完全匹配，例如：

```text
https://www.example.com/
```

### 7.2 创建 Google Cloud 项目

1. 打开 [Google Cloud Console](https://console.cloud.google.com/)。
2. 点击页面顶部的项目选择器。
3. 点击“新建项目”。
4. 输入项目名，例如：

```text
Search Index Submitter
```

5. 创建并切换到该项目。

### 7.3 启用 Search Console API

1. 打开“API 和服务”→“API 库”。
2. 搜索：

```text
Google Search Console API
```

3. 进入详情页并点击“启用”。

### 7.4 配置 OAuth 同意屏幕

1. 打开“API 和服务”→“OAuth 同意屏幕”或“Google Auth Platform”。
2. 填写应用名称、用户支持邮箱和开发者联系邮箱。
3. 个人用户通常选择“外部”。组织内部使用可根据 Google Workspace 情况选择“内部”。
4. 如果应用仍处于测试状态，把自己的 Google 账号添加为测试用户。
5. 添加 Search Console 所需权限范围：

```text
https://www.googleapis.com/auth/webmasters
```

Google 后台名称和菜单可能调整，以实际页面为准。

### 7.5 创建桌面 OAuth 客户端

1. 打开“API 和服务”→“凭据”。
2. 点击“创建凭据”→“OAuth 客户端 ID”。
3. 应用类型选择“桌面应用”。
4. 输入名称，例如：

```text
Search Index Submitter Desktop
```

5. 创建后点击“下载 JSON”。
6. 把文件保存到只有当前用户可以访问的位置，例如：

```text
Documents/search-index-submitter/google-client.json
```

### 7.6 软件内配置

打开“平台凭据”：

```text
Google OAuth 客户端 JSON：选择刚下载的 JSON 文件
Google Token 缓存：可以留空
```

第一次正式查询或提交时：

1. 浏览器会自动打开 Google 登录页；
2. 选择已经验证网站的 Google 账号；
3. 阅读授权信息；
4. 同意 Search Console 权限；
5. 完成后软件会保存授权 token，以后通常不必重复登录。

### 7.7 常见错误

| 提示 | 原因 | 处理方法 |
|---|---|---|
| access_denied | 账号未同意权限或不在测试用户中 | 加入测试用户并重新授权 |
| insufficient permission | 当前 Google 账号不是该站点所有者 | 在 Search Console 添加权限 |
| siteUrl not found | 软件站点地址和资源类型不匹配 | 检查协议、`www` 和域名资源 |
| redirect_uri_mismatch | OAuth 客户端类型选错 | 重新创建“桌面应用”客户端 |
| token 失效 | 用户撤销授权或 token 文件损坏 | 删除 token 缓存后重新登录 |
| quota exceeded | URL Inspection 查询额度用完 | 降低批量查询频率，稍后重试 |

官方资料：

- [Search Console API 授权](https://developers.google.com/webmaster-tools/v1/how-tos/authorizing)
- [创建 OAuth 客户端凭据](https://developers.google.com/workspace/guides/create-credentials#desktop-app)
- [提交 Sitemap API](https://developers.google.com/webmaster-tools/v1/sitemaps/submit)
- [URL Inspection API](https://developers.google.com/webmaster-tools/v1/urlInspection.index/inspect)

## 8. Bing Webmaster API Key 获取与配置

### 8.1 添加网站

1. 打开 [Bing Webmaster Tools](https://www.bing.com/webmasters/)。
2. 使用 Microsoft、Google 或支持的账号登录。
3. 可以从 Google Search Console 导入站点，也可以手动添加站点。
4. 按提示完成 DNS、XML 文件或 Meta 标签验证。

### 8.2 生成 API Key

1. 登录 Bing Webmaster Tools。
2. 打开右上角设置菜单。
3. 找到“API Access”或“API 访问”。
4. 首次进入时阅读并接受相关条款。
5. 打开“API Key”。
6. 点击“Generate API Key”。
7. 复制生成的 Key。

Bing API Key 通常按用户生成，不是按单个站点生成。同一账号下已经验证的多个站点可使用同一 Key。

### 8.3 软件内配置

在“平台凭据”中填写：

```text
Bing API Key：复制的完整 Key
```

建议同时配置 IndexNow。IndexNow 适合通知更新；Bing Webmaster API 适合查询和明确向 Bing 账号提交。

### 8.4 常见错误

| 提示 | 原因 | 处理方法 |
|---|---|---|
| HTTP 401/403 | API Key 无效或账号无站点权限 | 重新生成 Key，检查站点所有权 |
| site not found | 站点没有添加到 Bing | 先添加并验证站点 |
| URL 不属于站点 | 域名或协议不一致 | 按 Bing 中的站点地址输入 |
| 请求过多 | 超出平台频率或额度 | 减少重复提交，稍后重试 |

官方资料：

- [获取 Bing Webmaster API 访问权限](https://learn.microsoft.com/en-us/bingwebmaster/getting-access)
- [Bing Webmaster API 入门](https://learn.microsoft.com/en-us/bingwebmaster/getting-started)

## 9. Yandex OAuth Token 获取与配置

Yandex 可通过 IndexNow 接收 URL 更新；单独配置 Yandex OAuth 后，还可以通过 Webmaster API 提交 Sitemap。

### 9.1 添加并验证网站

1. 打开 [Yandex Webmaster](https://webmaster.yandex.com/)。
2. 登录 Yandex 账号。
3. 添加完整站点地址。
4. 通过 HTML 文件、Meta 标签或 DNS 完成验证。
5. 确认当前账号拥有该站点管理权限。

### 9.2 创建 Yandex OAuth 应用

1. 打开 [Yandex OAuth 应用管理](https://oauth.yandex.com/client/new)。
2. 创建新应用。
3. 填写应用名称，例如：

```text
Search Index Submitter
```

4. 选择适合本地客户端的应用类型或回调方式。
5. 添加 Yandex Webmaster 读写所需权限。页面名称可能随 Yandex 后台更新，请搜索 Webmaster 权限并同时允许读取与管理站点数据。
6. 保存后记录应用的 Client ID。

### 9.3 获取 OAuth Token

按 Yandex OAuth 页面提供的用户授权链接登录并授权应用。授权成功后获取 access token。

Token 必须属于已经验证目标站点的 Yandex 账号。

不要把 Yandex 密码填写到本软件中；软件只需要 OAuth Token。

### 9.4 软件内配置

在“平台凭据”中填写：

```text
Yandex OAuth Token：完整 access token
```

### 9.5 常见错误

| 提示 | 原因 | 处理方法 |
|---|---|---|
| HTTP 401 | Token 无效、过期或被撤销 | 重新授权获取 Token |
| 找不到站点 | 当前账号没有验证该域名 | 在 Yandex Webmaster 添加并验证 |
| 协议不匹配 | `http`、`https` 或 `www` 不一致 | 使用 Webmaster 中完全一致的地址 |
| 权限不足 | OAuth 应用缺少 Webmaster 权限 | 修改权限后重新授权 |

官方资料：

- [Yandex Webmaster API 授权](https://yandex.com/dev/webmaster/doc/en/tasks/how-to-get-oauth)
- [Yandex OAuth 介绍](https://yandex.com/dev/id/doc/en/concepts/ya-oauth-intro)

## 10. 360 搜索和神马搜索

目前工具没有把网页搜索结果抓取当作“官方收录状态”，也不会伪造 API 成功。

原因包括：

- 搜索结果受地区、账号、数据中心和时间影响；
- 搜索页面可能触发验证码或反自动化限制；
- 搜索结果中暂时找不到 URL，不代表平台数据库中一定不存在；
- 没有稳定公开 API 时，自动抓取容易误判。

请使用平台站长入口：

- 360 搜索站长平台：<https://zhanzhang.so.com/>

## 12. 首次使用标准流程

按照以下顺序操作：

1. 确认首页返回 HTTP 200。
2. 确认 HTTPS 证书有效。
3. 打开并检查 `robots.txt`。
4. 打开并检查 `sitemap.xml`。
5. 在需要的平台添加并验证站点。
6. 配置 IndexNow Key 文件。
7. 获取百度 token。
8. 获取 Bing API Key。
9. 创建 Google OAuth 客户端并完成首次授权。
10. 根据需要配置 Yandex OAuth Token。
11. 在软件中输入首页和新页面 URL。
12. 保持“提交前查重”开启。
13. 点击“预检”。
14. 处理所有失败和缺少凭据提示。
15. 点击“开始一键提交”。
16. 保存结果，用于后续排查和客户交付。

## 13. 如何判断是否配置成功

### IndexNow

- Key 文件公开地址返回 HTTP 200；
- 提交返回 HTTP 200 或 202；
- 没有出现 Key 与主机不匹配。

### 百度

- 返回成功推送数量；
- 能看到今日剩余额度；
- 没有 token 或 site 错误。

### Google

- 首次运行能正常打开授权页面；
- 授权后生成 token 缓存；
- URL Inspection 能返回覆盖状态；
- Sitemap 提交没有权限错误。

### Bing

- GetUrlInfo 能返回 URL 状态；
- URL 批量提交返回成功；
- 当前 API Key 可用于账号下的已验证站点。

### Yandex

- 能读取当前用户和已验证站点；
- 能找到完全匹配的站点地址；
- Sitemap 提交返回成功。

## 14. 安全建议

1. 不要把配置文件上传到网盘公开链接或 GitHub。
2. 不要使用来源不明的“共享 token”。
3. Google OAuth JSON、授权 token、百度 token、Bing Key 和 Yandex Token 都应视为敏感配置。
4. 发现泄露后立即到对应平台撤销并重新生成。
5. 公共电脑使用完毕后删除本机配置和 Google token 缓存。
6. 不要为了“更快收录”频繁重复提交相同 URL。
7. 不要提交垃圾页面、桥页、采集内容或违规内容。
8. 定期检查 Sitemap 是否仍然返回有效 XML。

## 15. 常见问题

### 提交成功后多久收录？

没有固定时间。可能是几分钟、几天，也可能最终不收录。内容质量、站点信誉、内部链接、服务器稳定性和平台策略都会影响结果。

### 为什么查重显示“无法确认”？

常见原因包括没有配置查询凭据、平台没有公开精确查询 API、接口额度用完或暂时网络异常。“无法确认”不会被当成“已收录”。

### 为什么 Google 没有逐个提交普通 URL？

Google 已停用旧 Sitemap Ping；Google Indexing API 也不是普通网站的通用提交接口。本工具遵循官方方式，通过 Search Console 提交 Sitemap，并用 URL Inspection 查询状态。

### IndexNow 是否包含 Google 和百度？

不包含。IndexNow 只通知参与该协议的搜索引擎。

### 同一个网站可以反复提交吗？

可以在内容真正新增或更新时提交。没有变化时不要高频重复提交。本工具默认会先查重，但并非所有平台都能精确查询。

### 可以一次提交很多网站吗？

可以。每行输入一个域名或 URL。需要注意百度 token 的站点映射，以及每个网站都要部署 IndexNow Key 文件。

### 软件会上传网站源码吗？

不会。工具只向搜索平台发送站点 URL、Sitemap 地址和平台要求的认证信息。
